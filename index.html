How It Works

This offline-only browser-based tool uses algorithms that produce a highly randomized pair of alphanumeric characters. Each pair represents 1 character of your private key. It doesn't double your security, it multiplies it by 3.101Ã—10^97. It makes your encoded keys unreadable and exponentially increases your backup security.

This mapping is output in CSV format and used to encode private keys. The encoded keys are stored in a separate file. It's an ultra-secure way to produce a backup since neither file on its own reveals anything about the original data.

Optionally you can encrypt the CSV using a password, using 256-bit AES encryption. These files should be split across multiple USB sticks and stored in various safe, known locations. Create copies for redundancy. This way, if one or more of the USB sticks are lost or damaged, you can still recover your private keys.

Only by recombining the cipher map and your encoded keys can you decode them back to their original form. Not only does it make your key storage highly resistant to theft, it also makes it easy to recover your keys in the event of a disaster.

This tool is designed to be used offline. It's a simple, secure way to protect your private keys.
